# Entity Records

Create one file per entity with stable ID in filename.

## Foundation requirement (startup gate)
Before chapter outlining, maintain a minimum launch set:
- 1 protagonist (`char_####`)
- 1 opposing force (`char_####` or `fac_####`)
- 2 launch locations (`loc_####`)
- 1 active plotline (`plot_####`)

## Suggested folder layout
- `canon/entities/characters/`
- `canon/entities/locations/`
- `canon/entities/plots/`
- `canon/entities/factions/`

Example filename pattern:
- characters/char_0001_name.md
- locations/loc_0001_name.md
- plots/plot_0001_name.md

Each entity file should include:
1. canonical facts
2. motivation or system function
3. timeline-relevant state
4. linked event IDs
5. open tasks or unknowns
